### 암호 알고리즘

### 개인키 암호화 기법
    - 동일한 키로 `암호화 하고 복호화 한다`
    - 속도가 빠르다
    - `대칭암호기법` 또는 `비밀키 암호화 기법`

### 개인키 암호화 기법의 종류
- 블록 암호화 방식
    - 한번에 하나의 데이터 블록을 암호화 하는 방식
    - 종류 : DES, SEED, AES, ARIA, IDEA
- 스트림 암호화 방식
    - `평문과 동일한 길이`의 스트림을 생성 비트단위 암호화
    - LFSR,RC4 등

### 공개키 암호화 기법
- 공개키는 `사용자에게 공개`, 복호화한 비밀키는 `관리자가 비밀리`에 관리
- 암호화 대상이 N개일때 2N이다.
- `비대칭 암호 기법` 이라고도 함
- 대표적으로 `RSA`기법
- 키의 분배가 용이, 관리해야 할 키의 개수 적음

### ARIA - 한국이 만듬
- 국가정보원과 산학연합회가 개발한 블록암호화 알고리즘
- 크기는 128 | 키길이에 따라 128,192,256 구분

### DES - NBS 미국이 만듬
- 64비트 암호화 알고리즘

### AES -> DES의 발전
- 한계를 느끼고 NIST가 `2001년` 발표한 `개인키 암호화`

### RSA -> 사람이름
- 1978년 MIT학생 `공개키`

### ECC -> RSA 발전
- 1985년 타원곡선으로 옮겨 효율성 높임

### Rabin
- 1979년 미하엘 라빈
- `소수분해`

### Hash
- `고정된 길이의 값이나 키로 변환하는것`
- `복호화가 거의 불가능` 일방향 함수
- 종류 : SHA 시리즈, MD4, MD5, N-NASH, SNEFRU

### 솔트
- 원문에 무작위값을 덧붙임

### 서비스 공격 유형

### 서비스 거부 공격의 개념
> 유형은 총 6개

- 서버 자원을 `고갈시킬` 목적으로
- 다수의 공격자 or 대량의 데이터 한곳에 집중적으로 전송함
- `서버의 정상 기능을 방해하는것`

### Ping of Death
- ICMP 패킷의 크기를 `프로토콜 허용범위 이상`으로 전송해 `마비`시킴
- 정의 구체화 밑에꺼까지 같이 적기
- Ping of Death 공격은 정상 크기보다 큰 ICMP 패킷을작은 조각(Fragment)으로 쪼개어 공격 대상이 조각화된 패킷을 처리하게 만드는 공격 방법이다.
### SMURFING
- IP나 ICMP의 특성을 악용해
- 엄청난 데이터의 양을 `한 사이트에 집중적으로 보냄`
- 네트워크 불능
- Smurf 공격은 `네트워크 라우터의 브로드캐스트(Broadcast) 주소`를 활용한 DoS 공격입니다

### SYN FLooding
- `가상의 클라이언트`로 위장하여
- `3-hand-shake`과정을 의도적으로 중단 시킴

### Land
- 송수진 주소를 자신으로 바꿔
- `자신에게 무한히 응답`하게 만듦

### DDos
- 여러곳에 분산된 공격지점에서 
- `한곳의 서버에 대해 분산 서비스 공격을 수행`

공격용 툴의 종류 : TrinOO, TFN, TFN2K, Stacheldraht

### Phishing -> 보이스피싱 생각
- 공기관, 금융기관을 `사칭`하여 정보 빼냄

### Ping Flood
- 특정사이트에 `매우많은 ICMP 메세지를 보내`
- 응답에 시스템자원을 모두 사용하게함

### Switch Jamming
- 위조된 접근 제어(MAC) 주소를 지속해서 네트워크로 흘려보냄
- 스위치 MAC주소의 저장기능을 환란시켜
- `더미허브처럼 작동`

### 블루 프린팅
- 블루투스 장비 공격

### Worm
- 연속적으로 자신을 `복제`
- 시스템 부하를 높여 다운되게만듬

### Key Logger Attack
- 키보드 움직임 탐지해 해킹
- 클릭형 키보드로 하게하는 보안 생각

### Ransomware - 파일 암호화
- 컴퓨터에 잠입해 내부 문서,파일 등을 암호화해
- 사용자에게 해독제를 조건으로 돈을 요구함

### DoS(Denial of Service) 공격과 관련한 내용으로 틀린 것은?
> 이걸 틀렸어? 야

1) Ping of Death 공격은 정상 크기보다 큰 ICMP 패킷을작은 조각(Fragment)으로 `쪼개어 공격` 대상이 조각화된 패킷을 처리하게 만드는 공격 방법이다.
2) Smurf 공격은 멀티캐스트(Multicast)를 활용하여 공격대상이 네트워크의 임의의 시스템에 패킷을 보내게 만드는 공격이다.
3) SYN Flooding은 존재하지 않는 클라이언트가 서버별로한정된 접속 가능 공간에 접속한 것처럼 속여 다른 사용자가 서비스를 이용하지 못하게 하는 것이다.
4) Land 공격은 패킷 전송 시 출발지 IP 주소와 목적지 IP주소 값을 똑같이 만들어서 공격 대상에게 보내는 공격방법이다.

[정답:2번]
Smurf 공격은 `네트워크 라우터의 브로드캐스트(Broadcast) 주소`를 활용한 DoS 공 격입니다

### Secure 코딩에서 입력 데이터의 보안 약점과 관련한 설 명으로 틀린 것은?
1) SQL 삽입 : 사용자의 입력 값 등 외부 입력 값이 SQL 쿼리에 삽입되어 공격
2) 크로스사이트 스크립트 : 검증되지 않은 외부 입력 값에의해 브라우저에서 악의적인 코드가 실행
3) 운영체제 명령어 삽입 : 운영체제 명령어 파라미터 입력값이 적절한 사전검증을 거치지 않고 사용되어 공격자가 운영체제 명령어를 조작
4) 자원 삽입 : 사용자가 내부 입력 값을 통해 시스템 내에사용이 불가능한 자원을 지속적으로 입력함으로써 시스템에 과부하 발생

[정답:4번]
경로 조작 및 자원 삽입은 데이터 입·출력 경로를 조작하여 서버 자원을 수정 및 삭제할 수 있는 보안 약점입니다.

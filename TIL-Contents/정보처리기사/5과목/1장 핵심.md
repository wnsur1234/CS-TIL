### CASE란?
> 자동화된 기법들을 말함, 소프트웨어 기능 향상에 도움을 줌

### 기능
- 생명주기 전 단계 연결
- 모순 검사 및 오류 검증
- 자료 흐름도 작성

### CASE의 원천 기술
- 프로토 타입
- 구조적 기법 : 구조적은 처리 기반
- 분산처리
- 자동프로그래밍
- 정보 저장소

---

### LOC 기법
> 기간 산정 기법

- 핵심 키워드 
    - 총코드줄/노력 => 총코드줄/인원*기간

### 수학적 산정기법 종류
- COCOMO 모형
- Putman 모형
- 기능점수(Fountion Point)모형

#### COCOMO 기법
- Bohem 형이 제안한것으로 LOC에의한 산정 기법
- 비교적 작은규모를 통계 분석함
- 필요한 노력(Man-Month) -> (인원 기간)

#### COCOMO소프트웨어 개발 유형
> 코드라인 : 조직형(5만) < 반분리형(30만 이하) < 내장형(30만 이상)

- 조직형 : 중소규모
- 반분리형 : 트랜잭션 처리, DB 관리시스템
- 내장형 : 초대형 프로젝트 규모

#### PutMan 모형
- 노력의 `분포`를 가정해줌
- Rayleigh-Norden 곡선

#### FP - 가중치 증대 요인
- 자료입력
- 정보충력
- 명령어
- 데이터파일
- 필요한 외부 루틴과의 인터페이스

### 자동화 추정 도구 - `SLIM`
Rayleigh-Norden 곡선과 Putman 예측 모델을 기초로 하는 자동화 도구

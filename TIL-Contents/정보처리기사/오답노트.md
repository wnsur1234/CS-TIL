### 익스트림 프로그래밍(XP)
> 익스트림 프로그래밍(eXtreme Programming)은 애자일 개발 방법론을 기반으로 하는 소프트웨어 개발 모형입니다.

- 소규모 개발 조직이 불확실하고 변경이 많은 요구를 접하였을 때 적절한 방법이다.
- 익스트림 프로그래밍을 구동시키는 원리는 상식적인 원리와 경험을 최대한 끌어 올리는 것이다.
- 구체적인 실천 방법을 정의하고 있으며, 개발 문서보다는 소스 코드에 중점을 둔다.

### 요구사항 분석에서 비기능적(Nonfunctional) 요구에 대한 설명으로 옳은 것은?
> `정답` : <u>‘차량 대여 시스템이 제공하는 모든 화면이 3초 이내에 사용자에게 보여야 한다’는 비기능적 요구이다.</u>

-> 비기능 요구사항 중 성능 요구사항에 해당합니다.

1) 시스템의 처리량(Throughput), 반응 시간 등의 성능 요구나 품질 요구는 비기능적 요구에 해당하지 않는다.
    - 성능 요구나 품질 요구는 비기능 요구사항에 해당합니다.   
3) 시스템 구축과 관련된 안전, 보안에 대한 요구사항들은 비기능적 요구에 해당하지 않는다.
    - 안전이나 보안에 대한 요구사항은 비기능 요구사항에 해당합니다.
4) ‘금융 시스템은 조회, 인출, 입금, 송금의 기능이 있어야 한다’는 비기능적 요구이다.
    - 기능 요구사항에 해당합니다.

내가 헷갈렸던 이유 : 비기능적? 직관적으로 정말 비기능을 말하는거 아닌가? 
-> 비기능적 요구 개념 확인 필요
> 비기능적 요구사항이란 : 시스템이나 프로젝트 개발 과정 등에서 지켜야할 `제약사항`을 의미함

4) 액터가 인식할 수 없는 시스템 내부의 기능을 하나의 유스케이스로 파악해서는 안된다.

### 소프트웨어 공학에서 워크스루(Walkthrough)에 대한 설명으로 틀린 것은?

1) 사용사례를 확장하여 명세하거나 설계 다이어그램, 원시코드, 테스트 케이스 등에 적용할 수 있다.

3) 인스펙션(Inspection)과 동일한 의미를 가진다. -> 틀림
4) 단순한 테스트 케이스를 이용하여 프로덕트를 수작업으로 수행해 보는 것이다.

### 단위 테스트(Unit Test)와 관련한 설명으로 틀린 것은?
1) 구현 단계에서 각 모듈의 개발을 완료한 후 개발자가 명세서의 내용대로 정확히 구현되었는지 테스트한다.
2) 모듈 내부의 구조를 구체적으로 볼 수 있는 구조적 테스트를 주로 시행한다.
3) 필요 데이터를 인자를 통해 넘겨주고, 테스트 완료 후 그결과값을 받는 역할을 하는 가상의 모듈을 테스트 스텁(Stub)이라고 한다.
4) 테스트할 모듈을 호출하는 모듈도 있고, 테스트할 모듈이 호출하는 모듈도 있다

[정답:3번]
③번은 `테스트 드라이버(Test Driver)에 대한 설명`입니다. 테스트 스텁(Test Stub)은제어 모듈이 호출하는 타 모듈의 기능을 단순히 수행하는 도구로, 일시적으로 필요한 조건만을 가지고 있는 시험용 모듈입니다. 테스트 드라이버와 테스트 스텁의 개념을 정확히 기억하고, 나머지 보기를 통해 단위 테스트의 특징을 정리하세요

### Stub VS Driver (단골 문제임)
> 일단 얘네는 둘다 `단위테스트` 안에 속한 도구

### 테스트 스텁 (Stub)
- 정의 
    - 테스트할 모듈이 호출하는 `하위 모듈을 대신`하는 가짜 모듈
    - 아직 구현되지 않은 하위 모듈 역할

- 언제 쓰나?
    - Top-Down 방식
    - `상위 모듈 테스트 시 하위 모듈이 없을 때`

### 테스트드라이버(Driver)
- 정의
    - 테스트할 모듈을 호출하는 가짜 상위 모듈
    - 아직 구현되지 않은 상위 모듈 역할

- ✔ 언제 쓰나?
    - Bottom-Up 방식
    - 하위 모듈부터 테스트할 때

### 정리
[Stub] 👉 “하위모델 대체” / 불린다
[Driver] 👉 “상위모델 대체” / 부른다

### 정규화 
> 릴레이션으로 표현할 수 있게 단계별로 분해 하는것

정규화 과정
> 비정규 릴레이션 -> 제 1정규형(1NF) -> 제 2정규형(2NF) -> 재 3정규형(3NF) -> BCNF -> 제 4정규형(4NF) -> 제 5정규형(5Nf)

- 비정규 릴레이션
- 제 1정규형 
- 도메인이 원자값
    - 반복속성 x
    - 다중값 X
- 제 2정규형
- 부분 함수적 종속성제거
    - 기본키 일부에만 종속 X
- 제3정규형 (3NF)
- 이행적 함수적 종속 제거
    - 키 → 일반속성 → 일반속성 ❌
- BCNF
- 결정자가 다 후보키여야
    - 결정자 ≠ 후보키 ❌

### 데이터베이스의 트랜잭션 성질들 중에서 다음 설명에 해당하는 것은?
1) Atomicity = 원자성
    - 완료(Commit)되든지 아니면전혀 반영되지 않도록 복구(Rollback)되어야 한다는 특성
2) Consistency = 일관성
    - DB 트랜잭션으로 트랜잭션 수행 전/후에 항상 올바른 상태 유지
3) Isolation = 격리성,독립성?
    - 여러 트랜잭션이 동시에 실행돼도 
    - 서로 간섭하지 않은 것처럼 보이게
4) Durability = 지속성
    - Commit이 완료된 데이터는
    - 시스템 장애가 나도 사라지지 않음

[정답:1번]
트랜잭션의 연산은 데이터베이스에 모두 반영되도록 완료(Commit)되든지 아니면전혀 반영되지 않도록 복구(Rollback)되어야 한다는 특성은 원자성(Atomicity)입니다.

### DDL,DCL,DML에 대하여

- DDL : 테이블 `구조 자체를 바꿈`
    - CREATE
    - ALTER
    - DROP : 테이블 자체를 삭제함 / 단, 외래키 존재 시 삭제 불가
    - TRUNCATE : 전체데이터 삭제(구조유지) / 전체 행 삭제
    - DROP TABLE ... CASCADE : 외래키까지도 테이블 자체를 삭제함 / 강제 삭제
- DCL : 데티어 제어
    - GRANT
    - REVOKE
- DML : 조회,수정,삭제,삽입
    - SELECT
    - INSERT
    - UPDATE
    - DELETE : 일부/전체 행 삭제
- TCL : 트랜잭션 제어어
    - COMMIT
    - ROLLBACK
    - SAVEPOINT

### 데이터 모델의 구성 요소 중 데이터 구조에 따라 개념 세계나 컴퓨터 세계에서 실제로 표현된 값들을 처리하는 작업을의미하는 것은?
1) Relation = 릴레이션 / `'테이블임'`
2) Data Structure = 데이터구조 / `어떤형태로 구성되어있는지`
3) Constraint = 제약조건 / `지켜야할 규칙`
4) Operation = 연산 / `'처리한다'`

### UDT와 TCP 프로토콜
> UDP와 TCP는 모두 전송 계층(Transport Layer)

- 역할 차이
    - TCP: 신뢰성
    - UDP: 속도

### UDP의 핵심 특징
- 비연결형
- 신뢰성 ❌
- 순서 보장 ❌
- 오류 제어 거의 없음
- 오버헤드 작음
- 빠른 전송
- 트랜스포트 계층

### TCP의 역할
> 신뢰성 있는 데이터 전송

- 하는 일
    - 연결 설정 (3-way handshake) / TCP연결 전 `3번의 메세지 교환`으로 통신준비확인 하는 것
    - 데이터 순서 보장
    - 오류 발생 시 재전송
    - 흐름 제어
    - 혼잡 제어

### IP의 역할
> 주소 지정 + 경로 설정(라우팅)

- 하는 일
    - 출발지 IP / 목적지 IP 지정
    - 패킷을 어느 경로로 보낼지 결정

### 소프트웨어 개발에서 정보 보안 3요소는?
> `정보 보안`의 3대 요소에는 `기밀성, 무결성, 가용성`

- 기밀성 : 인가된 사용자에 대해서만 자원 접근이 가능하다
- 무결성: 인가된 사용자에 대해서만 자원 수정이 가능하며 전송 중인 정보는 수정되지 않는다.
- 가용성: 인가된 사용자는 가지고 있는 권한 범위 내에서 언제든 자원 접근이 가능하다.

### 어떤 외부 컴퓨터가 접속되면 접속 인가 여부를 점검해서 인가된 경우에는 접속이 허용되고, 그 반대의 경우에는 거부할 수 있는 접근제어 유틸리티는?

1) tcp wrapper
    - 호스트 기반 접근제어 도구
    - 외부에서 서비스 접속 시 -> 허용 / 차단 여부를 판단
2) trace checker : 추적·분석 도구
    - 시스템 동작이나 이벤트 추적
    - 로그 분석, 추적 검사 목적
3) token finder : 인증 보조 도구 / `차단을 직접 수행하지는 않음`
    - 토큰(Token) 검색
    - 인증 정보, 키 값 등을 찾는 용도
4) change detector : 무결성 관리 도구
    - 파일·시스템 변경 감지
    - 무결성 검사

[정답:1번]
외부 컴퓨터의 접속 여부를 제어할 수 있는 접근제어 유틸리티는 TCP Wrapper입니다.

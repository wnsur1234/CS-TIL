# 데이터 베이스 엔진이란?
> `스토리지엔진`이라고도 부름

- 데이터베이스 관리 시스템(DBMS)이 데이터베이스에 대해 
- 데이터를 `삽입, 추출, 업데이트 및 삭제`하는데 사용하는 `기본 소프트웨어 컴포넌트`이다.
- 대부분의 데이터베이스 관리시스템은 DBMS의 사용자 인터페이스를 통하지 않고, 
- 사용자가 내장된 엔진과 상호작용을 할 수 있는 자신만의 애플리케이션 프로그래밍 인터페이스(API)를 포함

[최종 데이터 베이스의 흐름]
- 개발자/애플리케이션 ↔ (JDBC 드라이버, MySQL 프로토콜로 통신) ↔ MySQL 서버 (SQL 인터페이스 + 파서/옵티마이저/실행기)<br>
↔ (스토리지 엔진 API로 호출) ↔ InnoDB (스토리지 엔진: 실제 저장/조회 + 트랜잭션/락/MVCC/외래키 등 담당)

---

### 위 내용의 정리

---
[첫번째]
> 데이터를 `삽입, 추출, 업데이트 및 삭제`하는데 사용하는 `기본 소프트웨어 컴포넌트`이다.

여기서 말하는 CRUD는 여러분이 만드는 `REST CRUD API가 아니라`<br>
`INSERT / SELECT / UPDATE / DELETE 같은` SQL을 실제로 `디스크/메모리에 반영`하는 일

- 인덱스(B+Tree) 관리
- 데이터 페이지 읽기/쓰기(버퍼풀)
- 트랜잭션(Undo/Redo), MVCC, Lock
- 장애 시 복구(redo log 기반)
- 같은 DB 내부의 물리적인 저장/조회/갱신을 말합니다.

<u>즉, “CRUD 기본 컴포넌트”는 InnoDB가 테이블 데이터를 어떻게 저장하고 가져오고 갱신할지 담당한다는 의미입니다.</u>

---
[두번째]
> 사용자가 내장된 엔진과 상호작용을 할 수 있는 자신만의 애플리케이션 프로그래밍 인터페이스(API)를 포함

사용자가 자신만의 API로 엔진과 상호작용”에서 ‘사용자’는 개발자/앱(클라이언트) 쪽<br>
그 문장의 ‘사용자’는 보통 DBMS를 사용하는 애플리케이션/개발자를 뜻합니다.

```
다만 중요한 건:
개발자가 InnoDB에 직접 API 호출을 하는 경우는 거의 없고,
보통은 MySQL 서버가 제공하는 인터페이스를 통해 접근합니다.
```

- 그 “API”라는 표현은 보통 이런 걸 말합니다:
    - MySQL 프로토콜 기반 클라이언트 라이브러리: JDBC/ODBC, MySQL Connector 등
    - SQL 인터페이스(쿼리 요청 자체)
    - DBMS가 외부에 제공하는 접근 수단 전반

<u>즉, 문장 의미는:<br>
“앱(개발자)은 JDBC 같은 인터페이스로 MySQL에 요청하고, MySQL이 내부에서 InnoDB를 써서 실제 CRUD를 수행한다”
에 가깝습니다.<br>
</u>

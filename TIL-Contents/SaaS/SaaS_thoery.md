# SaaS란 무엇인가??
> Software as a Service의 약자로, 우리말로는 서비스형 소프트웨어라고 해요. 쉽게 말하면, 설치하지 않고 인터넷을 통해 바로 사용할 수 있는 소프트웨어를 말합니다.

### 예를들어..
- 카카오워크
- 네이버 웍스
- 구글드라이브
- MS Office 365
- Notion

즉, 예전에는 문서작성 프로그램을 컴퓨터에 직접 설치 해야 했다면, 지금은 인터넷만 있으면 `웹 브라우저에서 바로 사용할 수 있는 서비스`

### SaaS의 주요 특징
- 설치 불필요 
    - 브라우저만 있으면 사용 가능
- 언제 어디서나 접근 가능 
    - 인터넷만 있으면 OK
- 자동 업데이트 
    - 사용자가 직접 업데이트할 필요 없음
- 월정액 또는 연정액 과금  
    - 소프트웨어를 구입하는 것이 아니라 구독하는 형태

### SaaS를 왜 쓸까?

- 기업 입장: 관리가 쉬움, 비용 절감, 확장성 뛰어남
- 개인 사용자: 접근이 간편하고 최신 기능을 항상 사용 가능

> 그렇다면 뭐 Notion이나 구글드라이브같은거다 라는건 알겠어

### 의문점
> 근데 그럼 앱을 개발하듯 Notion같은 애플리케이션을 개발하고 이걸 웹브라우저 처럼 사용할 수 있도록 하는건가? 

### 알고보니...

✅ 그래서 최종 정리!

단순히 앱을 만들고 배포했다고 해서 SaaS는 아니고,
**사용자가 “자기만의 서비스처럼 쓸 수 있는 구조”와
그것을 가능하게 하는 기능들(결제, 권한, 멀티테넌시 등)**이 갖춰졌을 때 비로소 SaaS라고 부릅니다.

📌 한 줄 요약:

SaaS = 웹앱 + 사용자별 개인화 + 구독 결제 + 서비스화
단순 앱과의 차이는 바로 이 서비스화 수준에 달려 있어요.

즉, 단순애플리케이션과의 차이가 있는데 
이는 유료서비스로 개인화가 되냐 안되냐의 차이

🧱 추가로: 멀티테넌시까지 고려하면?

당신이 지금 설명하신 구조는 기본적으로 싱글테넌시 or 논리적 멀티테넌시에 가까워요.

하나의 DB에 사용자 데이터를 PK로 나누는 구조죠.

**진짜 고도화된 SaaS에서는 멀티테넌시(Multi-Tenancy)**를 더 명확하게 고려하기도 해요:

테넌시별 DB를 아예 나누거나,

워크스페이스 개념을 도입해서 팀 단위로 분리하는 방식 등

하지만 지금 단계에선 지금 설명하신 구조만으로도 충분히 SaaS입니다.
